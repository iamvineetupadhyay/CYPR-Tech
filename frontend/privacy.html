Privacy Policy & Data Architecture

Last Updated: July 18, 2026

CYPR Tech ("Company", "we", "us", or "our") is dedicated to protecting user privacy, minimizing data exposure, and enforcing secure data governance models under applicable data privacy laws, including the General Data Protection Regulation (GDPR) and the California Consumer Privacy Act (CCPA). This document outlines our technical data architecture, telemetry scopes, and volatile processing policies.

Section 1: Zero-Knowledge Password Isolation Architecture

Technical Declaration: The Platform does not ingest, log, or persist raw plain-text credentials in any permanent storage layer.

To ensure maximum data protection during Password Leak Analysis scans:

  • In-Flight Isolation: Submitted passwords are processed temporarily within transient active memory (RAM) segments of the Java Spring Boot runtime to calculate length, casing patterns, special character distributions, and overall entropy.
  • Masking & Hashing: Before the database writes any execution records to the Scan History logs in PostgreSQL, the plain-text input is dynamically masked and replaced with non-reversible masked characters (e.g., Password Check: ********).
  • Permanent Isolation: Under no circumstances are raw, unhashed user passwords recorded in application trace files, system logs, or PostgreSQL databases. Registered user credentials used for logging in are asymmetrically encrypted using Bcrypt cryptographic hashing algorithms.

Section 2: Volatile In-Memory File Processing Sandbox Policy

When files are submitted to the Malware Analysis Gateway, the following Zero-Disk Persistence policy is strictly enforced:

  1. Transient Buffers: Uploaded binaries, scripts, or documents are streamed directly into volatile Spring Boot RAM buffers. No local files are written to hard drives or persistent block storage.
  2. Feature Extraction: The heuristics engine evaluates entropy levels, section layouts, imports, and metadata inside this volatile container space.
  3. Immediate Purge: As soon as the analysis reports are compiled and sent as a JSON response, the RAM allocation is terminated, and the file data is purged completely from the server memory using system garbage collection routines. The only data persisted is the metadata (hash values, detected threat types, and size metrics) required to maintain your account's dashboard history.

Section 3: Log Analytics & Telemetry Scope

We collect and store technical telemetry metrics (including IP addresses, connection location, client User-Agent strings, and timestamp structures) strictly for security auditing, rate limiting verification, and concurrent session tracking.

DATA MONETIZATION GUARANTEE: CYPR Tech does not sell, lease, commercialize, license, or trade your telemetry logs, usage metadata, or account profiles with third-party data brokers, advertising networks, or any other commercial entities.

Section 4: User Data Access, Modification & Erasure Rights (GDPR / CCPA)

We respect the legal rights of individuals over their personal data. If you are registered on the Platform, you have the right to request:

  • Access and Export: Access to all personal profile details, settings preferences, and scan history logs.
  • Modification: Real-time correction of incorrect email addresses, usernames, or phone numbers in your Settings drawer.
  • Absolute Erasure ("Right to be Forgotten"): Upon initiating a profile delete request, all corresponding user tables, audit trace logs, and session tokens in our PostgreSQL databases are permanently purged within 7 business days, save for records required for compliance or rate limit abuse enforcement.
